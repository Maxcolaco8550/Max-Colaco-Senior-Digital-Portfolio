**1/12/26**

Today I realized that my games kept crashing when I would open them. I would open the game, and it would say press a button to configure. After a few seconds, it would send me back to the game selection screen. I did a little research because I was confused, and I found that a lot of potential crashes are caused by using the wrong emulator. I went into the settings and set my emulator to the MAME 2003 version. MAME 2003 is the best emulator to handle older games such as GALAGA and PACman.

**1/13/26**

Today, my teacher gave me a SANWA jLF-tP-8yt Joystick that I planned to use. I spent most of today doing research on how to code this type of joystick and what pin goes to what prong of the joystick. I learned that the different pins referred to a different direction as well as a ground pin. When programming it, I would need to make sure each direction had a different GPIO pin and a different key attached to it.

**1/15/26**

I continued to make sure the joystick worked, and since I didn't have the buttons yet, I turned to working on the design files. I began by tracing the outer panels that were created in the tutorial I found. I turned back to the design files whilst I waited for the buttons, and I created the center monitor. I matched the dimensions to the side panels so it would fit, and I added 2 rectangles for speakers. I would later delete these squares because I didn't like the way they looked.

**1/20/26**

Today I finally received the retro arcade buttons that I had ordered. I decided to work on coding the buttons to work today. I first began by cutting 22-gauge wire and stripping it to use on the button ends. I cut a piece of the red wire and used that to measure the black wire to ensure they were the same size. As soon as I began to solder the wire onto the prong of the button, I realized that since the prongs were metal and the wire was metal, it was going to be difficult to solder them together. Class had ended, so I no longer had time to work on it today.

**1/21/26**

When I came back to the buttons, I stubbornly began to solder them onto the prongs again. I did realize this time that if I melted solder onto the tip of the soldering iron, then I could place that solder on the metal, and they would stick together. I also realized that if I curled the wire around the prong, it would be easier to keep the wire in place. I spent class today finding that solution and soldering one of the buttons.

![GPIO Layout]({{ site.baseurl }}/assets/images/IMG_1920_(1).jpg)

**1/22/26**

Since I had so much trouble soldering one of the buttons, I decided to use one of them for proof of concept, and then got back and soldered them all. The next step was to place the case around the button and begin to program one of them, as well as the joystick. I had to install a GPIO Controller Driver, which allows each direction to be registered to a GPIO pin. I used the mk_arcade_joystick_rpi. I also had to make sure that it would install when booting up retropie so it would register. I had to add a note to the modules document in order to have the driver recognized and thus open when booting up the Raspberry Pi.
I finally was able to add each GPIO pin for each direction. I had to use 4 different pins for the 4 different directions. The pins I used are in the photo below.
